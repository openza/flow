# Reviewing Pull Requests

Openza Flow provides a streamlined interface for managing your PR review workflow.

## PR Inbox Overview

When you open Openza Flow, you'll see your PR inbox organized into two tabs:

| Tab | Sections |
|-----|----------|
| **Review Requests** | Pending Reviews, Recently Reviewed |
| **Created** | Open Pull Requests, Recently Created |

## Viewing PR Details

Each PR card shows:
- PR title
- Repository name (owner/repo format)
- PR number and branch info
- Author information with avatar
- Labels with color coding
- Time since last update
- Draft status (if applicable)

<Aside type="tip">
  Click on any PR card to open it directly on GitHub for a full review.
</Aside>

## Quick Actions

<Steps>
1. **Open PR** - Click the PR card to open it in your browser
2. **Search** - Use the search bar to find specific PRs
3. **Refresh** - Click the refresh button to fetch latest data
4. **Toggle Theme** - Switch between dark and light mode
</Steps>

## Search

Use the search bar in the header to filter PRs:
- Search by PR title
- Search by repository name
- Search by author
- Uses GitHub's search syntax

## Workflow Tips

1. **Check Review Requests first** - Address PRs waiting for your review
2. **Use notifications** - Desktop notifications alert you to new review requests
3. **Regular refresh** - Openza Flow auto-refreshes every 5 minutes, but you can trigger manual refresh anytime
4. **Track your PRs** - Use the Created tab to monitor feedback on your own PRs
