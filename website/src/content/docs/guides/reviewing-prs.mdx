# Reviewing Pull Requests

GitDesk provides a streamlined interface for managing your PR review workflow.

## PR Inbox Overview

When you open GitDesk, you'll see your PR inbox organized into sections:

| Section | Description |
|---------|-------------|
| **Awaiting Review** | PRs where your review is requested |
| **Your PRs** | Pull requests you've created |
| **Recently Reviewed** | PRs you've recently reviewed |

## Viewing PR Details

Click on any PR to see:
- PR title and description
- Author information
- Review status (pending, approved, changes requested)
- CI/CD status
- Files changed count

<Aside type="tip">
  Use the **Open in Browser** button to quickly jump to the PR on GitHub for a full review.
</Aside>

## Quick Actions

<Steps>
1. **View PR** - Click the PR card to expand details
2. **Open in Browser** - Click the external link icon
3. **Mark as Reviewed** - Update your local tracking
4. **Refresh** - Pull latest PR status
</Steps>

## Filtering PRs

Use the filter options to:
- Show only PRs in specific repositories
- Filter by review status
- Sort by date or priority

## Workflow Tips

1. **Start with oldest reviews** - Clear your backlog systematically
2. **Use notifications** - Enable desktop notifications to catch new review requests
3. **Regular refresh** - GitDesk auto-refreshes, but you can trigger manual refresh
